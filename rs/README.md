# RS-485 Microservice

## Директории

Директория `api` содержит `.proto`-файлы драйвера и микросервиса.

Директория `client` содержит gRPC-клиенты драйвера и микросервиса.

Директория `common` содержит общие типы, обработку ошибок и вспомогательные функции.

Директория `service` содержит реализацию микросервиса.

Директория `scripts` содержит скрипты сборки и очистки проекта.

## Сборка

### Linux

Сборка выполняется с помощью скрипта:

```bash
./scripts/build.sh
```

### Windows

В директории, содержащей `CMakeLists.txt`, выполните команды:

```bash
mkdir build
cmake -B build
cmake --build build
```

## Очистка

### Linux

Очистка выполняется с помощью скрипта:

```bash
./scripts/clean.sh
```

### Windows

Для очистки достаточно удалить директорию `build`.

## Запуск

### Linux

После сборки микросервис запускается командой:

```bash
./build/rs485_microservice
```

### Windows

После сборки запустите исполняемый файл:

```text
build\rs485_microservice.exe
```
