# RS-485 Microservice

gRPC-микросервис для взаимодействия с драйвером RS-485.

> **Проект предназначен для сборки и запуска под Linux (Ubuntu 24.04 или совместимые дистрибутивы).**

## Содержание

* Структура проекта
* Требования
* Сборка
* Очистка
* Запуск
* Тестирование
* Генерация gRPC-кода

---

# Структура проекта

```text
rs/
├── api/                 # .proto-файлы драйвера и микросервиса
├── client/              # gRPC-клиенты драйвера и микросервиса
├── common/              # Общие типы, обработка ошибок и вспомогательные функции
├── config/              # Конфигурационный файл микросервиса
├── fake_driver/         # Тестовая реализация драйвера RS-485
├── service/             # Реализация микросервиса
├── test_client/         # Консольный клиент для тестирования
├── scripts/             # Скрипты сборки, запуска и очистки
├── build/               # Рабочая сборка (создаётся автоматически)
├── build-test/          # Тестовая сборка (создаётся автоматически)
├── CMakeLists.txt
└── README.md
```

# Требования

Для сборки проекта необходимы:

* Linux (Ubuntu 24.04 или совместимый дистрибутив);
* CMake 3.16 или новее;
* компилятор с поддержкой C++17;
* Protobuf;
* gRPC;
* yaml-cpp;
* pkg-config.

Установка зависимостей:

```bash
sudo apt update

sudo apt install \
    build-essential \
    cmake \
    pkg-config \
    protobuf-compiler \
    libprotobuf-dev \
    libgrpc++-dev \
    protobuf-compiler-grpc \
    libyaml-cpp-dev
```

Проверить установку:

```bash
cmake --version
protoc --version
pkg-config --modversion grpc++
```

# Сборка

Все команды выполняются из директории `rs`.

```bash
cd rs
./scripts/build.sh
```

Во время сборки автоматически:

* генерируется gRPC-код из `.proto`-файлов;
* создаётся директория `build`;
* выполняется конфигурация CMake;
* собирается исполняемый файл `rs485_microservice`.

---

# Очистка

Для удаления результатов рабочей сборки выполните:

```bash
./scripts/clean.sh
```

---

# Запуск

После успешной сборки рекомендуется запускать микросервис с помощью скрипта:

```bash
./scripts/run.sh
```

Скрипт автоматически:

* устанавливает путь к конфигурационному файлу;
* запускает микросервис.

Перед запуском микросервиса должен быть запущен драйвер RS-485.

Для остановки микросервиса нажмите:

```text
Ctrl + C
```

# Тестирование

Проект поддерживает **два способа тестирования**.

## 1. Интеграционное тестирование через GUI

Основной способ проверки работы микросервиса.

Во время тестирования используется следующая схема:

```text
GUI
    │
    │ gRPC
    ▼
RS-485 Microservice
    │
    │ gRPC
    ▼
RS-485 Driver
```

Если реальный драйвер недоступен, вместо него может использоваться тестовая заглушка (`rs485_fake_driver`).

Через графический интерфейс можно проверить:

* отправку данных (`SendData`);
* подписку (`Subscribe`);
* получение данных;
* обработку ошибок;
* завершение подписки при отключении драйвера.

---

## 2. Тестирование без GUI

Для автономного тестирования предусмотрены два компонента:

* **fake_driver** — тестовая реализация драйвера RS-485;
* **test_client** — консольный gRPC-клиент микросервиса.

Во время такого тестирования используется следующая схема:

```text
test_client
      │
      │ gRPC
      ▼
RS-485 Microservice
      │
      │ gRPC
      ▼
fake_driver
```

Такой режим позволяет тестировать микросервис без запуска графического интерфейса.

---

## Сборка тестовой версии

Для сборки тестовых компонентов выполните:

```bash
./scripts/build_tests.sh
```

Будет создан каталог:

```text
build-test/
```

со следующими исполняемыми файлами:

```text
rs485_microservice
rs485_fake_driver
rs485_test_client
```

---

## Запуск тестовой версии

В первом терминале запустите тестовую реализацию драйвера:

```bash
./build-test/rs485_fake_driver
```

Во втором терминале запустите микросервис:

```bash
./scripts/run_tests.sh
```

или

```bash
RS485_CONFIG_PATH="$PWD/config/config.yaml" \
./build-test/rs485_microservice
```

После запуска микросервиса можно использовать один из двух клиентов:

* графический интерфейс;
* консольный `rs485_test_client`.

# Генерация gRPC-кода

Генерация gRPC-кода выполняется автоматически во время сборки.

Если были изменены `.proto`-файлы, достаточно повторно выполнить:

```bash
./scripts/build.sh
```

Сгенерированные файлы располагаются в каталоге:

```text
build/generated/
```

и не должны редактироваться вручную.
