# Инфраструктура

## Директории

Директория ```api``` содержит интерфейсы, по которым генерируется код gRPC.

Файл ```api/service_driver.proto``` содержит API для драйвера и микросервиса.

Файл ```api/ui_service.proto``` содержит API для UI и микросервиса.

Директория ```driver``` содержит код заглушку драйвера для тестирования.

Директория ```service``` содержит основной код микросервиса.

Директория ```ui``` содержит код пользовательского интерфейса.

## Сборка

### Linux

Полная сборка выполняется с помощью скрипта ```build.sh```.

### Windows

В директории, где находится файл ```CMakeLists.txt``` вызываются команды, аналогичные скрипту ```build.sh```.

```bat
mkdir build
cmake -B build
cmake --build build
```

## Очистка

### Linux

Полная очистка результатов сборки выполняется с помощью скрипта ```clean.sh```.

### Windows

Для очистки достаточно удалить директории ```build```, ```grpc_service_driver``` и ```grpc_ui_service```.

```bat
rmdir build grpc_service_driver grpc_ui_service
```
