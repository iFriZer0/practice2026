## Зависимости

### Linux

Для сборки достаточно установить следующие зависимости:

1. Qt6;
2. qmake6;
3. pkg-config;
4. CMake;
5. Make;
6. g++;
7. gRPC;
8. дополнительные библиотеки.

#### Qt6

Библиотеки Qt6 можно установить с помощью пакетного менеджера.

```bash
sudo apt-get install qt6-base-dev
```

#### qmake

Систему сборки qmake можно установить с помощью пакетного менеджера.

```bash
sudo apt-get install qmake6
```

#### pkg-config

Утилиту pkg-config можно установить с помощью пакетного менеджера.

```bash
sudo apt-get install pkg-config
```

#### CMake

Систему сборки CMake можно установить с помощью пакетного менеджера.

```bash
sudo apt-get install cmake
```

#### Make

Систему сборки Make можно установить с помощью пакетного менеджера.

```bash
sudo apt-get install make
```

#### g++

Компилятор g++ можно установить с помощью пакетного менеджера.

```bash
sudo apt-get install g++
```

#### gRPC

Библиотеки gRPC собираются из исходного кода.
Для этого нужно клонировать официальный репозиторий.

```bash
git clone --recurse-submodules -b v1.82.0 --depth 1 --shallow-submodules https://github.com/grpc/grpc
```

В переменную окружения ```MY_INSTALL_DIR``` нужно поместить директорию, в которую будут устанавливаться библиотеки gRPC.

Для сборки выполняются следующие команды.

```bash
cd grpc
mkdir -p cmake/build
pushd cmake/build
cmake -DgRPC_INSTALL=ON -DgRPC_BUILD_TESTS=OFF -DCMAKE_CXX_STANDARD=17 -DCMAKE_INSTALL_PREFIX=$MY_INSTALL_DIR ../..
make -j 4
make install
popd
```

В переменную окружения ```PATH``` нужно поместить директорию ```$MY_INSTALL_DIR/bin``` с бинарными файлами gRPC.

В переменную окружения ```PKG_CONFIG_PATH``` нужно поместить директорию ```MY_INSTALL_DIR/lib/pkgconfig``` с конфигурационными файлами библиотек.

#### Дополнительные библиотеки

Для работы gRPC обращается к дополнительным библиотекам, которые устанавливаются вместе с библиотеками самого gRPC во время сборки из исходного кода.
Однако утилита pkg-config не сможет обнаружить некоторые библиотеки, так как по умолчанию не для всех библиотек создаются конфигурационные файлы.

Дополнительные библиотеки можно установить с помощью пакетного менеджера.

```bash
sudo apt-get install zlib1g-dev libre2-dev libssl-dev 
```

## Сборка

Полная сборка выполняется с помощью скрипта ```build.sh```. Исполняемый файл — ```src/build/out/ui```.

## Очистка

Полная очистка выполняется с помощью скрипта ```clean.sh```.

## Запуск

Запуск на выполнение программы выполняется с помощью скрипта ```run.sh```.
